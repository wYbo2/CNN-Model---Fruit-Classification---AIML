## Overview

This project provides a full end-to-end pipeline for training and evaluating a custom CNN image classifier. It was built to classify fruit images into three categories: **Banana**, **Dragonfruit**, and **Unknown**.

The pipeline consists of two scripts:

| Script | Role |
|---|---|
| `train_local.py` | Splits raw data, builds & trains the CNN, saves the model and training graphs |
| `model_testing.py` | Loads one or multiple saved models, evaluates them, and generates a visual benchmark report |

---

## Features

- 🗂️ **Auto dataset splitting** — automatically divides raw images into Train (80%) / Val (10%) / Test (10%)
- 🔀 **Rich data augmentation** — flipping, rotation, zoom, translation, hue, brightness, contrast, saturation
- 🧠 **Custom 4-block CNN** with L2 regularization and dropout
- ⚡ **Mixed precision training** (`mixed_float16`) for faster GPU performance
- 🔁 **Smart callbacks** — EarlyStopping (patience=8) and ReduceLROnPlateau (patience=4)
- 💾 **Model saving** in `.h5` format
- 📊 **Training graphs** — accuracy and loss curves saved as PNG
- 🏆 **Multi-model benchmarking** — compare all `.h5`/`.keras` models side by side
- 🔥 **Overfitting diagnosis** — train vs. val accuracy gap analysis with color-coded alerts
- 🗺️ **Confusion matrices** — per-model heatmaps for all classes

## Dataset Setup

Create a `raw_data/` folder in the project root, with one **subfolder per class**:

```
raw_data/
├── banana/
│   ├── img_001.jpg
│   ├── img_002.jpg
│   └── ...
├── dragonfruit/
│   ├── img_001.jpg
│   └── ...
└── unknown/
    ├── img_001.jpg
    └── ...
```

> ✅ Supported formats: `.jpg`, `.jpeg`, `.png`  
> 📌 Recommended: at least **100–200 images per class** for meaningful training

To add or change classes, update this list in `train_local.py`:

```python
target_classes = ["banana", "dragonfruit", "unknown"]
```

The `processed_dataset/` folder is **automatically generated** when you run `train_local.py` — do not create or modify it manually.

## Usage

### 1. Training

```bash
python train_local.py
```

**What it does, step by step:**

1. Deletes any old `processed_dataset/` to ensure a fresh split
2. Splits `raw_data/` into `train/`, `val/`, `test/` (80/10/10)
3. Loads datasets with `image_size=(224, 224)` and `batch_size=32`
4. Applies data augmentation on-the-fly during training
5. Builds and compiles the CNN model
6. Trains for up to 50 epochs with EarlyStopping and ReduceLROnPlateau
7. Saves the trained model as `Jan_23_model2.h5`
8. Saves training accuracy/loss curves as `training_graph2.png`

**Expected terminal output:**

```
TensorFlow Version: 2.x.x
✅ SUCCESS! Found GPU: /physical_device:GPU:0
Splitting images into Train (80%), Val (10%), Test (10%)...
✅ Data split successfully!
Classes found: ['banana', 'dragonfruit', 'unknown']
Building Model...
Starting Training...
...
✅ Model saved as 'Jan_23_model2.h5'
✅ Graphs saved as 'training_graph.png'
```

---

### 2. Testing & Benchmarking

> ⚠️ **Prerequisite:** Run `train_local.py` at least once first to generate `processed_dataset/`.

```bash
python model_testing.py
```

**What it does:**

1. Scans the current directory and `models/` subfolder for all `.h5` and `.keras` files
2. Loads and evaluates each model against the validation set
3. Computes train accuracy, val accuracy, and the train/val **gap** (overfitting score)
4. Generates confusion matrices for each model
5. Renders and saves a full **visual benchmark dashboard** to `graphs/Report_YYYY-MM-DD.png`

**Console leaderboard example:**

```
MODEL NAME                     | VAL ACC  | STATUS
-----------------------------------------------------------------
Jan_23_model2.h5               | 94.3%    | OK
old_model_v1.h5                | 78.1%    | Overfit
baseline_model.h5              | 45.0%    | Underfit
```

**Status flags:**

| Status | Condition |
|---|---|
| `OK` | Val accuracy ≥ 50% and train/val gap ≤ 10% |
| `Overfit` | Train/val accuracy gap > 10% |
| `Underfit` | Val accuracy < 50% |

## Model Architecture

The CNN is a sequential 4-block architecture built with Keras:

```
Input: (224, 224, 3)
│
├── Data Augmentation (RandomFlip, Rotation, Zoom, Translation, Hue, Brightness, Contrast, Saturation)
├── Rescaling (1/255)
│
├── Conv Block 1:  Conv2D(32)  → MaxPool(2×2)
├── Conv Block 2:  Conv2D(64)  → MaxPool(2×2)
├── Conv Block 3:  Conv2D(128) → MaxPool(2×2)
├── Conv Block 4:  Conv2D(256) → MaxPool(2×2)
│
├── Flatten
├── Dense(256, relu) + L2 Regularization
├── Dropout(0.5)
│
└── Output: Dense(num_classes)   [raw logits]
```

- **Optimizer:** Adam (`lr = 1e-4`)
- **Loss:** `SparseCategoricalCrossentropy(from_logits=True)`
- **Regularization:** L2 (`λ = 1e-4`) on all Conv and Dense layers
- **Precision:** `mixed_float16` during training, `float32` during evaluation

## Training Configuration

All key hyperparameters are defined at the top of `train_local.py` and can be adjusted:

```python
IMG_SIZE        = (224, 224)   # Input image dimensions
BATCH_SIZE      = 32           # Training batch size
learning_rate   = 1e-4         # Adam optimizer learning rate
l2_strength     = 1e-4         # L2 regularization coefficient
epochs          = 50           # Max training epochs (EarlyStopping may stop earlier)

# EarlyStopping
patience        = 8            # Stop if val_loss doesn't improve for 8 epochs

# ReduceLROnPlateau
lr_factor       = 0.2          # Multiply LR by 0.2 on plateau
lr_patience     = 4            # Trigger after 4 stagnant epochs
min_lr          = 1e-6         # Minimum allowed learning rate

# Data split ratio
ratio           = (0.8, 0.1, 0.1)  # Train / Val / Test
```

## Benchmark Dashboard

The report generated by `model_testing.py` contains three sections:

```
┌─────────────────────────────────────────────────────────┐
│              Model Benchmark Report (timestamp)         │
├───────────────────────┬─────────────────────────────────┤
│  Leaderboard          │  Overfitting Diagnosis           │
│  (Val Acc bar chart)  │  (Train vs Val gap + color tag) │
├───────────┬───────────┼───────────┬─────────────────────┤
│ Model 1   │ Model 2   │ Model 3   │ Model 4  ...        │
│ Confusion │ Confusion │ Confusion │                     │
│ Matrix    │ Matrix    │ Matrix    │                     │
└───────────┴───────────┴───────────┴─────────────────────┘
```

Gap color coding on the overfitting chart:
- 🟢 **Green** — gap < 5% (well-generalized)
- 🟠 **Orange** — gap 5–10% (slight overfit)
- 🔴 **Red** — gap > 10% (overfit, consider more regularization)

## Known Limitations

- **3-class only by default** — adding new classes requires retraining from scratch
- **No test set evaluation** — `model_testing.py` uses the validation split; add test set eval manually if needed
- **No transfer learning** — a pretrained backbone (e.g., MobileNetV2, EfficientNet) would improve accuracy, especially with small datasets
- **Model filename is hardcoded** — `Jan_23_model2.h5` should be made configurable
- **Single GPU only** — no multi-GPU or distributed training support
